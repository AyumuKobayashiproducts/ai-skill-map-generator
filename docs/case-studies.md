## ケーススタディ集：このアプリが「刺さる」3つのシーン

> Findy などでプロジェクトを説明するときに、「どんな人がどう使うと価値が出るか」を具体的に語れるようにするためのメモです。

---

### Case 1: 駆け出しフロントエンドエンジニア（経験 1 年未満）

**Before**

- React / Next.js の個人開発はしているが、「自分の強み・弱みが分からない」  
- 転職サイトを見ても、どの求人に応募して良いか判断できない  
- 職務経歴書を書こうとしても、箇条書きが思いつかず手が止まる

**このアプリでやったこと**

- ホーム画面のサンプル文をベースに、自分の経歴に書き換えて診断  
- スキルマップで「Frontend が強く、Infra / AI が弱い」ことを可視化  
- Job Match 機能に、気になっているフロントエンド求人の JD を貼り付け  
- 1on1 練習モードで「自己紹介」「志望動機」を中心に 3 セッションほど練習

**After**

- フロントエンドを軸にしつつ、「Infra を Lv2 → Lv3 に上げる」など、具体的な 90 日プランができた  
- 求人マッチングのスコアを見ながら、「今すぐ応募できる求人」「半年後を目指す求人」を切り分けられた  
- 1on1 練習の総合スコアと履歴から、「最初は 2/5 → 練習後は 4/5 まで改善」と成長が数字で見えるようになった

**面接で話せるポイント**

- 「単に診断で終わらせず、**行動に落ちるレベルのプラン（30日 / 90日）** まで落とし込んだ」こと  
- 1on1 練習のスコア履歴を見せながら、「アウトプットを通じて改善していくプロダクト設計」の話ができる

### Case 2: フルスタック志望の 3 年目エンジニア

**Before**

- フロントエンド・バックエンドどちらも触っているが、「フルスタックとしてどこまで評価されるか」が分からない  
- スキルセットが広く薄くなっていて、何から伸ばすべきか迷っている  
- 自分の GitHub リポジトリをどうポートフォリオとして見せればいいか悩んでいる

**このアプリでやったこと**

- ゴールを「フルスタック」に設定してスキルマップを生成  
- Job Match で、フルスタック募集の JD を 2〜3 件貼り付けてマッチングスコアを比較  
- Career Risk で「属人化リスク」「自動化リスク」がどこにあるか確認  
- ポートフォリオ整理機能で、自分のリポジトリから案件 TOP3 のサマリーと Markdown を生成

**After**

- 「Backend / Infra をもう 1 段階伸ばすと、フルスタック求人とのマッチ度が一気に上がる」ことが明確になった  
- リスクレーダーを元に、「特定プロジェクト依存からの脱却」という学習テーマを設定できた  
- Markdown 出力をベースに、GitHub の README や Findy のプロジェクト欄を短時間で整えられた

**面接で話せるポイント**

- 「フルスタック志望を **“どのスコアをどこまで上げるか” という具体的指標** に落とし込んだ話」  
- 「ポートフォリオを手作業で書くのではなく、アプリから半自動で生成するワークフロー」を説明できる

### Case 3: 転職活動前に “市場価値” を確認したいバックエンドエンジニア

**Before**

- 自社サービスでバックエンド中心に 3 年ほど経験があるが、「社外でどれくらい通用するか」が不安  
- 面接で話せるエピソードがマンネリ化しており、うまく伝えられていない感覚がある  
- 現職に不満はないが、キャリアリスク（陳腐化や自動化）が気になっている

**このアプリでやったこと**

- バックエンド寄りの職務経歴を入力し、スキルマップと転職準備スコアを確認  
- 複数のバックエンド求人を Job Match に通し、「どの求人が今の自分に近いか」をスコアで比較  
- Career Risk で、自分のスキルセットがどの程度「陳腐化リスク」「自動化リスク」を抱えているかを数値で把握  
- 1on1 練習で、技術的なエピソードや失敗談を STAR 法で整理し直した

**After**

- 「今すぐ転職する／1〜2年準備する」のどちらも取りうる選択肢として、メリット・デメリットを整理できた  
- 求人マッチングとリスク分析を踏まえた **キャリアプラン（例：SRE 寄りに広げる）** をセットできた  
- 1on1 練習で作った模範回答をベースに、面接で話すエピソードが整理され、自信を持って話せるようになった

**面接で話せるポイント**

- 「市場とのギャップとリスクを定量的に見て、**転職を焦らず計画的に進める意思決定** をした話」  
- バックエンドエンジニアとして、**技術だけでなくキャリアリスクも含めて自分をモニタリングする習慣**を持っていることを示せる

---

これらのケーススタディは、README や Findy、面接時の「このプロジェクトで何を意識したか」を説明するときに、そのままエピソードとして利用できます。
