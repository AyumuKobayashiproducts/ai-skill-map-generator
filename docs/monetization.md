## 料金プラン・マネタイズのアイデアメモ

> このドキュメントは、実装済みのコードというより「もし本当に売るならどう課金するか？」を整理するためのメモです。  
> ポートフォリオや面接で、「プロダクトとしてどこまで考えているか」を説明する材料として使う想定です。

---

## 想定するユーザーと支払い主

- **個人ユーザー（1〜5年目くらいの Web エンジニア）**
  - 自分の転職活動・キャリア整理のために利用
  - クレジットカード / App Store / Play Store 決済などが想定

- **スクール / コミュニティ運営者**
  - 受講生のスキル棚卸し・面談用の補助ツールとして利用
  - 管理者アカウント + 受講生アカウントという構成

このプロジェクトではまず「個人ユーザー向けのサブスクリプション」を前提に設計を考えています。

---

## シンプルな料金モデル案

### Free プラン（現状の実装に近い状態）

- 月額: 無料
- 機能:
  - スキルマップ生成（回数制限あり：例: 月 3 回まで）
  - 学習ロードマップ / 求人マッチング / リスク診断（Free 回数の範囲内で利用可）
  - 1on1 練習（セッション数に制限：例: 月 3 セッションまで）
- 目的:
  - 「このツールが自分に合うか」を試すトライアル
  - 無料でも十分価値を感じられるが、「本気でやるなら Pro」というポジション

### Pro プラン（個人向け）

- 月額: 980〜1,500円 程度を想定
- 機能:
  - スキルマップ・求人マッチング・リスク診断の回数制限を大幅に緩和 or 無制限
  - 1on1 練習のセッション履歴をフルで保存（Free は直近数件のみ）
  - ダッシュボードでのスコア推移グラフ（転職準備スコア / 1on1 スコアなど）
  - ポートフォリオ出力テンプレートの追加（Findy / Green / レジュメ用 など）
- 価値:
  - 「転職活動の 1〜3 ヶ月間を本気で走り切るためのパートナー」として位置付け

### Team / Mentor プラン（将来案）

- 月額: 応相談 / シート課金
- 機能:
  - 管理者が受講生アカウントを複数管理
  - 受講生ごとのスキルマップ・1on1 履歴を一覧で閲覧
  - 面談メモの追加・エクスポート
- 用途:
  - プログラミングスクール、社内勉強会、メンタリングコミュニティなど

## 実装イメージ（簡易版）

このリポジトリでは、現時点では **実際の決済処理や有料プランの制限ロジックは実装していません**。  
代わりに、以下のような形で「あとから課金を組み込みやすい設計」を意識しています。

- Supabase のユーザープロファイルに `plan` フィールドを追加（`free` / `pro` など）
- API ルート側で `plan` を参照し、呼び出し回数や 1on1 セッション保存件数にソフトリミットを掛ける
- 有料決済は Stripe / Paddle などの外部サービスと連携し、Webhook で `plan` を更新する

このように、「課金まわりは別サービスに任せつつ、アプリ側はプラン単位の権限制御に集中する」構成を想定しています。

## 面接・ポートフォリオで話せるポイント

- 「無料で価値を感じてもらいつつ、本気ユーザーには Pro プランで課金してもらう」という、SaaS で一般的な構成に沿っていること
- データ構造や Usage ログ（`/admin/usage`）を活用して、どの機能が課金ポイントになり得るかを検証できる土台を作っていること
- 実装していない部分（決済 / Webhook / プラン管理）についても、どのレイヤーで何を担当させるかまでイメージできていること
